# Recast Getting Started — PAT & Plugin Setup

## Getting a PAT (API token)

1. Log in to the Recast app at https://app.getrecast.com.
2. Open the **account menu** in the top nav bar — the dropdown labeled with your email address.
3. Click **"Generate API Token"**.
4. Confirm the dialog. **Warning:** this revokes any existing API token for the account — if a token is already in use elsewhere (scripts, another machine, Claude Desktop), reuse that token instead of generating a new one.
5. The token is shown **exactly once** in a modal — copy it immediately. It cannot be viewed again, only regenerated.

## Configuring the plugin

1. Add the token to `~/.claude/settings.json` under the top-level `env` key, merging with any existing contents (create the file if it doesn't exist):

   ```json
   {
     "env": {
       "RECAST_PAT": "<the-token>"
     }
   }
   ```

   This is the recommended location: it reaches every way Claude Code runs — terminal, desktop app, and IDE extensions. A shell-profile export (`export RECAST_PAT=<the-token>` in `~/.zshrc` or `~/.bashrc`) only works for sessions launched from a terminal — the desktop app never reads shell profiles. Suggest the shell export *in addition* only if the user will run skill-generated scripts outside Claude Code.

2. Restart Claude Code — open a new session in the terminal, or fully quit and relaunch the desktop app.
3. On first use of a Recast MCP tool, a browser window opens for OAuth sign-in with the user's Recast account. This is separate from the PAT — both are required. Sign-in is occasional, not per-session.
4. Verify with `/mcp`: the `recast` server should show as connected. Don't judge by the desktop app's Customize/Connectors settings panels — they don't reliably reflect MCP status.

## One token, one name

Everything in this plugin reads the token from `RECAST_PAT` — the MCP server receives it as the `X-Recast-PAT` header, and code generated by the API skills reads the `RECAST_PAT` environment variable directly. The `env` block in settings.json covers both inside Claude Code; running generated scripts outside Claude Code additionally needs the shell export.

## Troubleshooting

- **Tools fail with auth/401 errors from the Recast API:** `RECAST_PAT` is unset, mistyped, or expired — check the `env` block in `~/.claude/settings.json` (or `echo $RECAST_PAT` for shell-export setups) — or the token was revoked by a later "Generate API Token" click.
- **Works in the terminal but not the desktop app:** the PAT is only in a shell profile, which the desktop app never reads. Move it to `~/.claude/settings.json` as above, then fully quit and relaunch the app.
- **Browser OAuth prompt reappears:** the OAuth session token expired — sign in again; this is expected occasionally.
- **No "Generate API Token" in the account menu:** the menu items are permission-gated per account — the user's access level doesn't allow personal access tokens. They should contact their Recast admin or CSM.
- **`recast` server missing from `/mcp`:** reinstall the plugin and restart Claude Code.
- Never print, log, or display the token.
